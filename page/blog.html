Discussing the scope of
Var Let Const

Var

1 : Variables are declared in JavaScript using var.

2 : Variables declared with var are function-scoped, meaning they are only accessible within the function in which they are declared or globally if declared outside of any function.

3 : They are also hoisted, which means they are moved to the top of their containing function or global scope during the compilation phase. This can sometimes lead to unexpected behavior.

Let

1 : let was introduced in ECMAScript 6 (ES6) to provide block-scoped variables.

2 : Variables declared with let are block-scoped, which means they are only accessible within the nearest enclosing block, such as a function, loop, or conditional statement.

3 : let variables are not hoisted to the top of their scope, and they have a more predictable behavior compared to var.

Const

1 : const is also introduced in ES6, and it is used to declare constants, which cannot be reassigned after they are defined.

2 : Like let, const is block-scoped, meaning it is only accessible within the nearest enclosing block.

3 : const variables must be initialized when declared and cannot be reassigned, although the value they hold can still be mutable if it's an object or array.

What does REST API mean?

A REST API (Representational State Transfer Application Programming Interface) is a standardized way for different software systems to communicate over the internet. It follows a set of principles, including using standard HTTP methods like GET, POST, PUT, and DELETE to perform actions on resources identified by URLs. REST APIs are widely used in web development for creating, accessing, and manipulating data on servers, making them a fundamental part of building modern web and mobile applications.
